# Умный Журнал

API для учебного журнала с ролями учителя и ученика. Ученики получают токены за подтвержденные активности и оценки, а затем могут тратить их на мерч.

## Технологии

- Go + Fiber
- PostgreSQL
- Ethereum adapter через go-ethereum
- Mock смарт-контракта для локальной разработки

## Основные сценарии

### Учитель

1. Авторизуется через общий endpoint `/auth/login`.
2. Просматривает учебные группы.
3. Создает предметы и группы.
4. Добавляет зарегистрированных учеников в группы.
5. Ставит оценки ученикам.
6. Просматривает оценки группы.
7. Подтверждает или отклоняет заявки учеников на дополнительные активности.
8. Начисляет токены вручную, если это нужно администратору журнала.
9. Работает только с группами и предметами, которые привязаны к его профилю.

### Ученик

1. Регистрируется через `/register`.
2. Авторизуется через `/auth/login`.
3. Просматривает свою группу и оценки группы.
4. Просматривает баланс токенов.
5. Покупает мерч за токены.
6. Отправляет дополнительную активность учителю на подтверждение.
7. Просматривает свои заявки на активности.
8. Просматривает историю покупок мерча.

## Настройка

Скопируйте `.env.example` и переименуйте в `.env`. Заполните вашими данными 

Если `RPC_URL`, `CONTRACT_ADDRESS` или `CONTRACT_ADMIN_PRIVATE_KEY` недоступны, приложение продолжит работу с mock-адаптером смарт-контракта.

## Миграции

В проекте есть SQL-миграции:

- `migrations/001_init.sql` создает таблицы пользователей, учителей, учеников, групп, предметов, оценок, достижений и мерча.
- `migrations/002_seed_teachers_and_merch.sql` добавляет сидовых учителей и товары мерча.
- `migrations/003_activity_status_teacher_scope_and_purchases.sql` добавляет статусы активностей, привязки учителей к группам/предметам и историю покупок.

Сидовые учителя авторизуются по email из миграции, пароль: `password`.
В текущем учебном режиме пароли хранятся в таблице `users` как обычный текст.

## Запуск

```bash
go run cmd/api/main.go
```

По умолчанию сервер слушает порт `3000`.

## API

OpenAPI-спецификация лежит в `docs/openapi.yaml`.
Curl-сценарии по основным юзкейсам лежат в `docs/curl_usecases.md`.

### Auth

- `POST /auth/login` — вход ученика или учителя.
- `POST /auth/logout` — выход.
- `GET /auth/session` — текущая сессия.

Пример входа:

```json
{
  "login": "anna.ivanova@school.edu",
  "password": "password"
}
```

### Ученики

- `POST /register` — регистрация ученика.
- `GET /students/group` — группа ученика и список одногруппников.
- `GET /students/grades` — оценки группы ученика.
- `GET /students/balance` — баланс токенов ученика.
- `GET /students/merch` — список мерча.
- `POST /students/merch/buy` — покупка мерча.
- `GET /students/purchases` — история покупок ученика.
- `POST /students/achievements` — заявка на дополнительную активность.
- `GET /students/achievements` — заявки ученика.

Пример регистрации:

```json
{
  "name": "Иван Петров",
  "email": "ivan.petrov@example.com",
  "group_id": 1,
  "password": "secret"
}
```

Пример заявки на активность:

```json
{
  "title": "Участие в олимпиаде",
  "description": "Решил региональный этап"
}
```

### Учителя

Все маршруты `/teachers/*` требуют роль `teacher`.

- `GET /teachers/groups` — список групп текущего учителя.
- `POST /teachers/groups` — создать группу и автоматически привязать ее к текущему учителю.
- `POST /teachers/groups/attach` — привязать существующую группу к текущему учителю.
- `POST /teachers/subjects` — создать предмет и автоматически привязать его к текущему учителю.
- `POST /teachers/subjects/attach` — привязать существующий предмет к текущему учителю.
- `POST /teachers/groups/add-student` — добавить ученика в группу.
- `POST /teachers/grades` — поставить оценку.
- `GET /teachers/groups/:id/grades` — оценки группы.
- `GET /teachers/achievements/pending` — неподтвержденные активности.
- `POST /teachers/achievements/confirm` — подтвердить активность и начислить токены.
- `POST /teachers/achievements/deny` — отклонить активность.
- `POST /teachers/tokens/award` — вручную начислить токены.

Пример создания группы:

```json
{
  "name": "БПИ-231"
}
```

Пример оценки:

```json
{
  "student_id": 1,
  "subject_id": 1,
  "value": 5
}
```

Пример привязки существующего предмета:

```json
{
  "subject_id": 1
}
```

## Текущие ограничения

- Баланс токенов хранится в PostgreSQL; смарт-контракт сейчас вызывается как внешний адаптер при начислении.
- Поле `confirmed` у активностей оставлено для обратной совместимости; основная логика использует `status`.
- История покупок хранит товар, цену на момент покупки и дату, но пока не содержит статуса выдачи мерча.

## Возможное развитие

- Админская роль для централизованного назначения учителей на группы и предметы.
- История изменения оценок.
- Интеграция с ИИ для персональных рекомендаций ученикам.
- NFT или бейджи за особые достижения.
